# ReNamer WPF - 功能测试指南

**更新时间**: 2026-02-13（与当前实现对齐）

## 🎯 测试目标

验证现代化 Fluent Design 界面的所有功能是否正常工作。

---

## 🚀 启动应用

```bash
cd ReNamerWPF
dotnet run --project ReNamer
```

或直接运行：
```bash
.\ReNamer\bin\Debug\net8.0-windows\ReNamer.exe
```

## ✅ 测试清单

### 1️⃣ 窗口控制（自定义标题栏）

**测试步骤**：
- [ ] 拖动蓝色标题栏，窗口应该跟随移动

- [ ] 点击「—」按钮，窗口应该最小化
- [ ] 点击「□」按钮，窗口应该最大化/还原

### 5️⃣ 添加规则功能

**测试步骤**：
- [ ] 点击规则面板工具栏的「+ 添加」按钮（或按 Ins）
- [ ] AddRuleDialog 对话框应该打开
- [ ] 左侧列表显示所有 17 种规则类型：
  ```
  Replace, Insert, Delete, Remove, Case, Serialize, Extension,
  Padding, Strip, CleanUp, Transliterate, RegEx, Rearrange,
  ReformatDate, Randomize, UserInput, Mapping
  ```

#### 5.1 测试 Replace 规则

- [ ] 选择「Replace」规则
- [ ] 右侧配置面板显示：
  - Find: 输入框
  - Replace with: 输入框
  - Case sensitive 复选框
- [ ] 分隔符按钮（|）与 Meta Tag 按钮可用
- [ ] 配置：Find=`test`, Replace=`file`
- [ ] 点击「Add Rule」按钮
- [ ] 规则应该出现在规则列表中：`☑ | Replace | "test" → "file"`

#### 5.2 测试 Serialize 规则

- [ ] 选择「Serialize」规则
- [ ] 配置：
  - Start: `1`
  - Increment: `1`
  - Padding: `3`
  - Type: `Decimal`
- [ ] 规则应该添加到列表：`☑ | Serialize | 001, 002, 003...`

## 🎓 测试技巧

1. **使用测试文件夹**：
   ```bash
   mkdir test_files
   echo "test1" > test_files/test1.txt
   echo "test2" > test_files/test2.txt
   echo "test3" > test_files/test3.jpg
   ```

2. **备份重要文件**：
   - 永远不要用真实文件测试重命名
   - 使用专门的测试目录

3. **验证 Undo 功能**：
   - 重命名后，菜单 → Undo Rename（Ctrl+Shift+Z）

4. **检查错误处理**：
   - 尝试重命名到已存在的文件名
   - 尝试重命名只读文件
   - 尝试使用非法字符（`<>:"/\|?*`）
