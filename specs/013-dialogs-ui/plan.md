# 实施计划：013 弹窗与规则配置面板 UI 优化

## 目标
统一弹窗与规则配置面板的层级、密度、按钮规格与文案资源化，保证与主界面设计方向一致。

## 范围
- AddRule/Settings/Filters 弹窗
- 规则配置面板（17 个）
- 文案资源与辅助菜单（RuleConfigHelper）

## 实施策略
1. 建立弹窗通用样式（按钮、分组标题、间距）
2. 统一弹窗布局密度与对比度
3. 规则配置面板统一边距与最小尺寸
4. 全量资源化描述性文案（XAML + RuleConfigHelper）
5. 复核并运行验证

## 风险与对策
- 风险：资源键缺失导致运行时错误  
  对策：新增键同时补齐中英文资源并运行验证
- 风险：布局收敛后内容拥挤  
  对策：保留 8px 基础间距，关键区域使用 12px

## 验收标准
- 弹窗与规则配置面板布局统一  
- 文案无硬编码英文  
- 编译通过且可启动运行
