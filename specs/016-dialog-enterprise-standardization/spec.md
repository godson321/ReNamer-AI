# 功能规格说明：对话框与规则面板企业化统一标准

**功能分支**：`016-dialog-enterprise-standardization`  
**创建日期**：2026-02-22  
**状态**：Draft  
**输入**：用户描述："把同一套企业化规则扩展到 SettingsDialog.xaml、TextInputDialog.xaml、各规则面板（*.xaml）统一骨架和按钮标准"

## 用户场景与测试（必填）

### 用户故事 1 - SettingsDialog 企业化统一（优先级：P1）

作为用户，我希望设置对话框与其他核心对话框视觉和布局一致，减少认知切换并提升产品专业感。

**优先级原因**：设置是高频入口，样式不统一会直接降低全局体验一致性。

**独立测试方式**：仅打开 `SettingsDialog`，确认其骨架、按钮、间距、标题层级符合企业化标准即可判定完成。

**验收场景**：

1. **Given** 打开设置对话框，**When** 查看结构，**Then** 为“顶部说明卡 + 中部内容卡 + 底部操作卡”。
2. **Given** 中文语言环境，**When** 查看底部按钮和标签，**Then** 不存在文本裁切或错位。

---

### 用户故事 2 - TextInputDialog 企业化统一（优先级：P1）

作为用户，我希望文本输入对话框与系统其他对话框在视觉和交互上保持一致，输入体验稳定且可预期。

**优先级原因**：文本输入对话框是通用交互组件，统一后可覆盖多处场景并降低维护成本。

**独立测试方式**：单独打开 `TextInputDialog`，验证骨架、输入区域、主次按钮样式与尺寸规则。

**验收场景**：

1. **Given** 打开文本输入对话框，**When** 查看输入框与提示区，**Then** 标签和输入控件对齐且间距一致。
2. **Given** 中文语言环境，**When** 查看按钮文案，**Then** 显示完整且点击区域满足最小尺寸标准。

---

### 用户故事 3 - 全部规则面板统一骨架与按钮标准（优先级：P1）

作为用户，我希望所有规则配置面板视觉规范一致，避免“每个面板像不同产品”的割裂感。

**优先级原因**：规则面板数量多、使用频繁，是一致性和可维护性的核心区域。

**独立测试方式**：逐一切换规则面板，检查布局骨架、标签列、输入高度、按钮样式、中文显示完整性。

**验收场景**：

1. **Given** 打开任意规则配置面板，**When** 查看布局，**Then** 标签列、输入控件高度和分组样式符合统一标准。
2. **Given** 面板包含按钮操作区，**When** 检查主次按钮，**Then** 使用统一企业按钮样式并满足中文优先尺寸。
3. **Given** 面板包含“数值 + 单位”输入行，**When** 检查单位下拉，**Then** 无箭头或文本遮挡。

---

### 边界与异常场景

- 窗口缩放到最小尺寸附近时，内容不得互相遮挡或超出可视区域。
- 主题切换后，卡片与文字对比度必须保持可读。
- 中文/英文资源切换后，不得出现按钮文本裁切或基线抖动。
- 历史面板局部自定义样式与通用样式冲突时，必须以通用标准优先并保留原业务逻辑。

## 需求（必填）

### 功能性需求

- **FR-001**：必须在 `DesignSystemResources.xaml` 提供并维护企业化通用样式集（卡片、输入控件、主次按钮）。
- **FR-002**：`SettingsDialog.xaml` 必须采用统一三段骨架（顶部说明卡 + 中部内容卡 + 底部操作卡）。
- **FR-003**：`TextInputDialog.xaml` 必须采用统一三段骨架并接入通用主次按钮样式。
- **FR-004**：所有规则面板 `Views/RuleConfigs/*.xaml` 必须统一基础布局规范（标签列宽、输入高度、分组留白、标题层级）。
- **FR-004A**：规则面板统一改造时不要求“顶部说明卡”，保留紧凑内容区与操作区即可；三段骨架要求仅适用于对话框（如 `SettingsDialog`、`TextInputDialog`）。
- **FR-005**：所有目标页面按钮必须按中文优先尺寸标准执行（`MinWidth >= 128`、`MinHeight >= 36`、`Padding=14,6`）。
- **FR-006**：所有目标窗口必须开启像素对齐（`UseLayoutRounding=True`、`SnapsToDevicePixels=True`）。
- **FR-007**：含“数值+单位”输入行的目标页面，单位列宽不得小于 `84`，防止箭头与文本遮挡。
- **FR-008**：主题相关画刷必须使用 `DynamicResource`，满足运行时主题切换稳定性要求。
- **FR-009**：本次改造不得改变规则执行、保存、加载等业务逻辑行为。

### 关键实体

- **DialogStyleSet（对话框样式集）**：统一企业化样式键集合，供 Settings/TextInput/规则面板复用。
- **PanelLayoutContract（面板布局契约）**：定义标签列宽、输入高度、按钮尺寸、间距与标题层级等硬性规范。
- **TargetPanelInventory（目标面板清单）**：本次纳入统一改造的页面列表（Settings、TextInput、全部 RuleConfigs）。

## 成功标准（必填）

### 可量化结果

- **SC-001**：`SettingsDialog` 与 `TextInputDialog` 100% 接入统一骨架和按钮样式。
- **SC-002**：`Views/RuleConfigs` 下全部在用面板（15 个）完成统一规范接入。
- **SC-003**：中文环境下主次按钮与单位下拉无裁切、无遮挡，人工回归通过率 100%。
- **SC-004**：完成一次 `dotnet build` 与一次 `dotnet run` 冒烟验证（存在文件锁时需记录并重试）。
- **SC-005**：主题切换后关键对话框和规则面板无 `UnsetValue` 相关显示异常。
